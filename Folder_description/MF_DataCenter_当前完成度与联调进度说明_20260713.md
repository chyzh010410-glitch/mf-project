# MF_DataCenter 当前完成度与联调进度说明

更新时间：2026-07-13  
负责人：MF_DataCenter

## 1. 当前结论

MF_DataCenter 已从基础运营后台推进为可用的“AI 运营与知识治理工作台”。

- **DataCenter 本体完成度：约 80%**。运营观察、数据治理、AI 会话沉淀、问题/样本运营、知识工作台和 MF_EP 草稿对接已具备可运行能力。
- **三项目受控联调完成度：约 65%**。`MF_AgentService -> MF_DataCenter` 与 `MF_AgentService -> MF_EP` 已完成真实调用验证；`MF_DataCenter -> MF_EP` 的草稿创建、发布和下线接口已完成代码对接，但尚未执行一次真实的“发布后检索、下线后失效”闭环验收。
- **生产上线完成度：约 50%**。仍缺发布后 AgentService 增量同步、完整回滚、三方自动化契约测试和生产配置/权限演练。

上述比例用于管理排期，不表示发布批准。

## 2. 已完成能力

### 2.1 运营与治理基础

- 数据源契约、字段契约、快照和质量问题治理。
- 指标字典、计算登记、日/小时快照刷新。
- 运营总览已呈现可信/风险状态、治理待办和通知入口。
- 商品、内容、商家、AI 分析页具备后端分页；左侧导航固定，不随主内容滚动。
- 前端分页已拆分为单表请求，避免翻页时重复加载整页分析数据。

### 2.2 AI 会话与运营闭环

- `dc_ai_conversation_log`：记录客服会话、意图、解决状态、满意度。
- `dc_ai_tool_call_log`：记录 Agent 工具调用链路与耗时。
- 未解决问题池：支持优先级、负责人、截止时间、处理结论和知识补充动作。
- 样本候选池：支持审核、批量通过、推荐入库、导出和后端分页。
- 用户会话历史接口：支持按用户和会话查询、软删除、鉴权与审计。
- AI 评测闭环、通知中心已具备基础数据结构和页面入口。

### 2.3 AI 知识运营工作台

- 从未解决问题池聚合知识缺口。
- 管理候选来源、可信度、AI 草稿和 AI 预审信息。
- 内容候选可创建 MF_EP 内部草稿；低风险候选可调用发布/下线接口，高风险候选禁止直接发布。
- 候选记录保存 `mf_ep_draft_id`、`mf_ep_content_id` 和失败原因，便于审计和重试。
- 优质回答沉淀已接入：审核通过且“推荐入库”的样本，可创建知识缺口和内容候选；正文直接取原样本回答。
- `source_sample_id` 唯一索引防止同一优质回答重复沉淀；候选预审字段保存样本、会话、来源、审核人和审核备注。

## 5. 未完成事项与优先级

### P0：完成一条正式受控发布闭环

选取低风险 FAQ，按以下步骤执行并记录审计结果：

```text
优质样本审核通过
-> DataCenter 创建内容候选
-> 创建 MF_EP 草稿
-> 发布到 MF_EP
-> AgentService 重新索引/增量同步
-> Agent 查询命中新内容
-> DataCenter 下线
-> Agent 查询确认不再命中
```

验收时不得使用高风险内容；发布测试内容应在下线后明确清理或保留为正式受控样本。

### P1：补发布后的 AgentService 同步

- 明确 AgentService 的同步接口鉴权和调用方。
- DataCenter 在 MF_EP 发布成功后调用同步接口，并保存同步结果与失败原因。
- 增加“已发布但未同步/同步失败”的治理任务或通知。

### P1：补三方自动化契约测试

- AgentService 写会话、工具调用、问题池、样本池的契约测试。
- DataCenter 调用 MF_EP 创建草稿、发布、下线的 MockServer 或测试环境测试。
- 发布后 AgentService 检索命中的端到端回归测试。

### P2：完善生产治理

- MF_EP 内容版本历史与回滚。
- 高风险内容人工复核人、复核意见和审批审计。
- 生产环境内部令牌托管、权限最小化、告警和失败重试策略。
