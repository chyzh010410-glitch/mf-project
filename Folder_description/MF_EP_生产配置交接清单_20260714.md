# MF_EP 生产配置交接清单

更新时间：2026-07-14  
适用范围：MF_EP `mf-fertilizer` 服务的生产环境配置与 AI 内容表迁移。

## 1. 已完成的 MF_EP 配置准备

- 已提供 `fertilizer-api/src/main/resources/application-prod.yml`。
- 生产 Profile 不包含数据库密码、Redis 密码或内部 token 的默认值，全部由部署环境变量提供。
- 生产 Profile 关闭 Knife4j。
- MF_EP 不配置、保存或使用大模型 API Key；该密钥只由 MF_AgentService 负责人管理。

## 2. 总负责人分发后由 MF_EP 注入的变量

| 环境变量 | 用途 | 来源/责任 |
| --- | --- | --- |
| `MF_DB_URL` | 生产 MySQL 连接地址 | 部署环境 |
| `MF_DB_USERNAME` | 数据库账号 | 部署环境 |
| `MF_DB_PASSWORD` | 数据库密码 | 部署环境/密钥托管 |
| `MF_REDIS_HOST` | Redis 地址 | 部署环境 |
| `MF_REDIS_PORT` | Redis 端口，默认 `6379` | 部署环境 |
| `MF_REDIS_PASSWORD` | Redis 密码 | 部署环境/密钥托管 |
| `MF_EP_INTERNAL_TOKEN` | DataCenter、AgentService 调用 MF_EP 内部接口的统一凭据 | 总负责人统一生成与分发，MF_EP 负责人注入自身服务 |
| `MF_DATACENTER_BASE_URL` | DataCenter 内部地址 | 部署编排 |
| `MF_DATACENTER_INTERNAL_TOKEN` | MF_EP 调用 DataCenter 的内部凭据 | 总负责人分发、DataCenter 负责人主责 |
| `MF_DATACENTER_IDENTITY_SECRET` | DataCenter 身份校验凭据 | 总负责人分发、DataCenter 负责人主责 |

不得将上述变量值写入 Git、`application-prod.yml`、前端配置或日志。`MF_EP_INTERNAL_TOKEN` 必须与 AgentService、DataCenter 使用的值一致。

## 3. AI 内容表迁移

在目标 `fertilizer` 库按以下顺序单次执行：

```text
1. fertilizer-api/src/main/resources/db/ai-content-draft-v1.sql
2. fertilizer-api/src/main/resources/db/ai-content-governance-v2.sql
3. fertilizer-api/src/main/resources/db/ai-content-governance-v3.sql
4. fertilizer-api/src/main/resources/db/ai-content-sync-governance-v4.sql
```

这些脚本为一次性数据库迁移，已执行的环境不得重复执行同一脚本。V1/V2 建立草稿、版本和同步事件基础结构；V3 补齐版本审核时间；V4 补齐同步失败回写与人工重试审计字段。

## 4. 启动方式

部署环境完成变量注入和迁移后，以 `prod` Profile 启动 MF_EP：

```text
SPRING_PROFILES_ACTIVE=prod
```

本轮按项目总负责人安排，不在各项目并行配置阶段执行跨服务接口测试。待三方完成各自生产配置后，由总负责人统一组织生产环境复验。

## 5. 边界说明

- MF_EP 负责正式内容、草稿发布/下线/回滚、同步事件与内部接口鉴权。
- AgentService 负责 AI 模型 API Key、事件消费、失败回写、索引验证与 ack。
- DataCenter 负责治理审计、候选编排和闭环脚本。
- 总负责人负责内部 token 的生成、分发、记录和三方部署编排。
