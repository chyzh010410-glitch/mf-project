# Debugging SOP

Use this when investigating failures or unexpected behavior.

1. Reproduce or locate the failure signal.
2. Capture the exact error, command, input, or user-visible symptom.
3. Trace from the failing boundary inward.
4. Check recent changes and relevant project notes.
5. Make the smallest fix that addresses the root cause.
6. Verify with the failing case first, then nearby regression coverage if practical.
7. Record reusable pitfalls only when the issue is likely to recur.
