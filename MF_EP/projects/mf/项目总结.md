# 苗丰施肥管控平台 — 项目总结

## 项目概述

面向种植户和园艺爱好者的垂直电商 + 内容社区平台。业务覆盖树苗/化肥商城交易、树木百科知识库、科普文章社区、AI 智能客服。管理后台供运营人员管控全流程，C 端供消费者浏览商品、下单购买、查阅百科、参与社区互动。

## 技术引入

### 并发与锁

| 技术 | 位置 | 用途 |
|------|------|------|
| `ReentrantLock` | `ClientOrderController.create()` | 下单时锁住库存扣减，防止并发超卖 |
| Redis `DECR` + `setIfAbsent` | `ClientOrderController.create()` | 原子扣库存 |
| Redis `SETNX` 分布式锁 | `OrderTimeoutTask.java` | 防止多实例重复执行定时任务 |

### 异步与定时

| 技术 | 位置 | 用途 |
|------|------|------|
| `@Async` + `ThreadPoolTaskExecutor` | `AsyncConfig.java` → `NotificationServiceImpl` | 下单后异步推送消息通知 |
| `@Scheduled` | `OrderTimeoutTask.java` | 每分钟扫描超时未支付订单自动取消 |

### 缓存

| 技术 | 位置 | 用途 |
|------|------|------|
| `@Cacheable` / `@CacheEvict` | `FertilizerServiceImpl`, `TreeServiceImpl`, `FertilizationRuleServiceImpl`, `ClientProductController` | 替代手动 Redis，Spring Cache + RedisCacheManager 三级缓存（30min/30min/24h/5min） |

### AOP 与上下文

| 技术 | 位置 | 用途 |
|------|------|------|
| `@Aspect` + `@Around` | `OperationLogAspect.java` | 自动记录管理员操作到 system_log 表 |
| `ThreadLocal` | `UserContext.java` | 消除 8 个 Controller 中重复的 JWT 解析代码 |

## AI 客服接入

MF_EP 客户端 `/ai` 页面通过 Vite 代理调用独立 Agent 服务：

```text
mf-frontend-client /ai
  -> /agent-api/api/agent/chat
  -> MF_AgentService http://localhost:8092
```

本项目不再包含 `mf-ai` 目录，不再启动 Python AI 服务。
