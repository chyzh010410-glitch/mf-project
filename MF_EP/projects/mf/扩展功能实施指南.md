## 1. 订单详情页

### 现状
- 后端 `ClientOrderController.detail()` 已完整实现，返回 `OrderVO`（含订单号、状态、金额、地址快照、商品明细列表）
- 前端 `order/detail.vue` 只有一行 `<p>订单加载中...</p>`，无 script

### 实施步骤

**文件:** `mf-frontend-client/src/views/order/detail.vue`

**1.1 数据获取**
```js
// 用 getOrderDetail(id) 从 API 模块拉数据
import { getOrderDetail, cancelOrder, confirmOrder } from '@/api/order'
const order = ref(null)
onMounted(async () => {
  const res = await getOrderDetail(route.params.id)
  if (res.code === 200) order.value = res.data
})
```

**1.2 页面结构**
- **订单状态栏**: 用 `el-steps` 展示 待付款→待发货→已发货→已完成 进度，当前状态高亮
- **收货地址卡片**: 解析 `order.addressSnapshot` JSON 字符串，展示收件人/电话/地址
- **商品明细表**: 遍历 `order.items`，列：商品名称、单价、数量、小计
- **金额汇总**: 商品总额 / 运费 / 优惠 / 实付金额
- **操作按钮**: 
  - `pending_pay` → "取消订单" + "去支付"按钮
  - `shipped` → "确认收货"按钮
- **订单信息**: 订单编号、创建时间、支付时间、物流单号

**1.3 状态映射**
```js
const statusMap = {
  pending_pay:  { text: '待付款', tag: 'warning', step: 1 },
  pending_ship: { text: '待发货', tag: 'primary', step: 2 },
  shipped:      { text: '已发货', tag: 'success', step: 3 },
  completed:    { text: '已完成', tag: 'success', step: 4 },
  cancelled:    { text: '已取消', tag: 'info',    step: 0 }
}
```

**1.4 顺手修订单列表页的状态标签**
`order/list.vue` 第15行 `{{ order.status }}` 处改为动态渲染：
```html
<el-tag :type="statusMap[order.status]?.tag">{{ statusMap[order.status]?.text }}</el-tag>
```

### 参考
- 后端返回数据结构: `OrderVO.java` (`fertilizer-common/.../vo/client/OrderVO.java`)
- API 函数: `order.js` 已导出 `getOrderDetail`, `cancelOrder`, `confirmOrder`

## 3. 支付流程

### 现状
- 后端 `Payment` 实体 + Service 完整，但无 Controller
- 订单创建时状态设为 `pending_pay`，但没有任何支付调用
- 管理端 "标记已付款" 是手动改状态，不是真支付
- 前端无支付页面

### 设计思路

考虑到这是肥料商城（非高频小额），采用 **模拟支付 + 管理员审核** 模式最务实，避免直接对接微信/支付宝的资质门槛。

### 3.1 后端 — 新建 ClientPaymentController

**文件:** `fertilizer-api/.../controller/client/ClientPaymentController.java`

```java
@RestController
@RequestMapping("/client/payment")
@RequiredArgsConstructor
public class ClientPaymentController {

    private final OrderEntityService orderService;
    private final PaymentService paymentService;

    // POST /client/payment/pay
    //   入参: { orderId, payMethod: "wechat"|"alipay"|"balance" }
    //   逻辑:
    //     1. 校验订单状态为 pending_pay
    //     2. 创建 Payment 记录 (status=pending)
    //     3. 模拟支付成功 → 更新 Payment.status=success
    //     4. 更新订单 status=pending_ship, payTime=now
    //     5. (可选) 扣除库存已在创建订单时扣了，这里不重复扣
    //     6. 返回支付结果

    // GET /client/payment/{orderId}  — 查询某订单的支付记录
}
```

### 3.2 后端 — 新增支付超时取消（可选）

在 `ClientOrderController` 上加一个定时任务概念：创建订单后 30 分钟未支付自动取消。可后续再实现自动定时，先支持手动取消 + 前端倒计时提示。

### 3.3 前端 — 订单详情页加"去支付"按钮

在订单详情页 (第1项) 的 `pending_pay` 状态下，增加"去支付"按钮：

```js
const handlePay = async (order) => {
  await ElMessageBox.confirm(`确认支付 ¥${order.payAmount}？`, '支付确认')
  const res = await request({ url: '/client/payment/pay', method: 'post', data: {
    orderId: order.id,
    payMethod: 'wechat'
  }})
  if (res.code === 200) {
    ElMessage.success('支付成功')
    order.status = 'pending_ship'  // 即时更新本地状态
  }
}
```

### 3.4 管理端 — 对接真实支付状态

- `Orders.vue` 的"标记已付款"改为调用 `/admin/orders/{id}/pay-confirm`（管理员确认收到款项）
- 如果后续对接真实微信支付，支付回调 URL 设为 `/client/payment/wechat-notify`

### 参考
- `ClientOrderController.create()` 中的订单创建流程，看 payAmount 的计算方式
- `AdminOrderController` 中的状态流转逻辑

## 4. 用户端高频三页（收藏 / 历史 / 地址）

这三个页面 **后端 API 全部完整**，纯前端填空。

### 4.1 我的收藏 — favorite/index.vue

**后端接口:**
| 接口 | 方法 | 说明 |
|------|------|------|
| `/client/favorites` | GET | 分页列表（type=product 筛选） |
| `/client/favorites` | POST | 添加收藏 `{targetType:"product", targetId}` |
| `/client/favorites/{id}` | DELETE | 取消收藏 |

**前端实现:**
- 商品卡片网格（复用首页 `ProductCard` 样式）
- 每个卡片：图片、名称、价格、"取消收藏"按钮
- 空状态：`<el-empty description="暂无收藏">`
- 分页器

**产品详情页加收藏按钮** — 在 `product/detail.vue` 的"立即购买"旁边加一个星标按钮。

### 4.2 浏览历史 — history/index.vue

**后端接口:**
| 接口 | 方法 | 说明 |
|------|------|------|
| `/client/history` | GET | 分页列表 |
| `/client/history` | DELETE | 清空全部历史 |

**前端实现:**
- 商品卡片列表，按时间倒序
- 顶部 "清空全部" 按钮（二次确认）
- 空状态提示

**记录浏览历史** — 需要在 `product/detail.vue` 的 `onMounted` 中加一行：
```js
request({ url: '/client/history', method: 'post', data: {
  targetType: 'product',
  targetId: product.value.id,
  targetName: product.value.name,
  targetImage: product.value.coverImage
}})
```

### 4.3 收货地址管理 — user/address.vue

**后端接口 (ClientUserController):**
| 接口 | 方法 | 说明 |
|------|------|------|
| `/client/addresses` | GET | 地址列表 |
| `/client/addresses` | POST | 新增地址 |
| `/client/addresses/{id}` | PUT | 编辑地址 |
| `/client/addresses/{id}` | DELETE | 删除地址 |
| `/client/addresses/{id}/default` | PUT | 设为默认 |

**前端实现:**
- 地址卡片列表（类似 checkout 页面的地址选择卡片样式）
- 每张卡片：收件人、电话、地区+详细地址、默认标签、"编辑"/"删除"操作
- "新增地址"按钮 → 弹窗表单（复用 checkout 页面的地址弹窗）
- 设为默认功能

> 可以把 checkout 页面的地址弹窗提取为独立组件 `AddressDialog.vue`，在 checkout 和 address 两个页面共用。

## 验证方案

### 订单详情页
1. 注册用户 → 浏览商品 → 立即购买 → 提交订单
2. 跳转到订单列表 → 点击"订单详情" → 确认显示完整信息
3. 测试"取消订单"按钮 → 确认状态变为已取消

### 管理端商品CRUD
1. 管理员登录 → 商品管理 → 新增商品（填写所有字段）
2. 列表搜索/筛选 → 编辑商品信息 → 上架/下架
3. 切换到用户端 → 确认新品出现在商城

### 支付流程
1. 创建订单 → 点击"去支付" → 确认弹窗 → 支付成功
2. 订单状态从"待付款"变为"待发货"
3. 管理员后台查看订单 → 确认支付状态

### 收藏/历史/地址
1. 商品详情页点收藏 → 到收藏页确认可见 → 取消收藏
2. 浏览商品 → 到历史页确认记录可见 → 清空历史
3. 地址管理页 → 新增/编辑/删除/设默认 → checkout 页确认默认地址生效
