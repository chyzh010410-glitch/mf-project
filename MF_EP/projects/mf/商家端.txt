任务：实现商家客户端 V1。

背景：
当前 mf 项目是苗木/肥料电商平台，已有：
1. C 端消费者端：mf-frontend-client
2. 管理端：mf-frontend
3. 后端：mf-fertilizer

当前架构更接近“消费者 + 管理员/运营”的平台自营模型，还没有真正的商家角色。
现在需要新增“商家”角色和商家端，采用“平台审核商家”模式：
- 商家注册/入驻后状态为 pending
- 管理员审核通过后状态为 approved
- 只有 approved 商家可以登录商家端
- 商家只能管理自己的商品、查看自己的订单商品、发货自己的订单

一、后端角色与鉴权

请在 RoleEnum 增加商家相关身份，例如：
- MERCHANT = "merchant"

要求：
- JWT 仍使用 userType 区分身份
- 商家登录成功后 userType=merchant
- /merchant/** 默认要求商家身份
- /merchant/auth/register 和 /merchant/auth/login 放行
- 商家 token 使用独立 RedisKey，例如 merchantToken(token)
- 不要复用 clientToken，也不要复用 admin loginToken

JwtInterceptorConfig 需要支持：
- /client/** 仍是消费者
- /admin/** 仍是管理员
- /merchant/** 是商家

UserContext 可增加：
- isMerchantUser()

二、商家领域

新增商家实体和相关 mapper/service/application service。

建议表名：
merchant

字段至少包括：
- id
- username
- password
- shop_name
- contact_name
- phone
- status
- audit_remark
- audit_time
- last_login_time
- create_time
- update_time
- deleted

状态：
- pending：待审核
- approved：已通过
- rejected：已拒绝
- disabled：已禁用

要求：
- 商家注册默认 pending
- 商家登录只允许 approved
- 密码处理方式尽量沿用现有 user/sys_user 的方式，不要引入新的复杂加密方案
- 注册时校验 username、phone 不重复

三、管理端商家审核

新增管理端接口：
- GET /admin/merchants：分页列表，可按状态/关键词筛选
- GET /admin/merchants/{id}：详情
- POST /admin/merchants/{id}/approve：审核通过
- POST /admin/merchants/{id}/reject：审核拒绝，支持审核备注
- POST /admin/merchants/{id}/disable：禁用商家

管理端前端 mf-frontend 新增菜单/页面：
- 商家管理
- 支持查看商家列表
- 支持通过/拒绝/禁用
- 显示状态中文标签

四、商品归属商家

给 product 增加 merchant_id 字段。

要求：
- 商家新增商品时，merchant_id 自动写当前商家 ID
- 商家编辑/删除商品时，必须校验 product.merchant_id == 当前商家 ID
- 商家商品列表只能看到自己的商品
- 管理端商品管理仍可查看全部商品
- C 端商品列表和详情继续正常展示，不需要强制显示商家信息，除非实现成本很低

如果现有 schema SQL 在 db 目录中，请补 migration/seed SQL。
如果没有统一 migration 机制，请新增一个清晰命名的 SQL 文件，例如：
mf-fertilizer/fertilizer-api/src/main/resources/db/merchant-v1.sql

五、订单项归属商家

给 order_item 增加 merchant_id 字段。

要求：
- 消费者下单创建订单项时，根据 product.merchant_id 写入 order_item.merchant_id
- 如果历史商品 merchant_id 为空，可以允许平台自营商品 merchant_id 为空
- 商家订单列表基于 order_item.merchant_id = 当前商家 ID 查询
- 商家不能看到不属于自己的 order_item

V1 不做复杂拆单：
- order 主表仍保持一个订单号
- 多商家拆单、多包裹、多结算留到以后
- 商家端订单详情只展示属于自己的商品项
- 商家发货时只允许处理自己相关的订单项
- 如果当前订单状态模型不支持“订单项级发货”，V1 可以在商家发货时更新主订单状态为已发货，但必须在代码注释或文档中说明这是 V1 简化策略

十一、手动验收场景

请按以下流程验证或说明如何验证：

1. 商家注册

- 不能登录商家端

2. 管理员审核

4. 商品管理
