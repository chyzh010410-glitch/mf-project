# Workspace Index

Use this file first to decide what to read. Do not load the full Workspace by default.

## Default Task Context

- Workspace rules: `README.md`
- Routing index: `index.md`
- Long-term user preferences: `preferences.md`
- Current project overview: `projects/<project>/brief.md`

## Where To Look

- Long-term personal preferences: `preferences.md`
- Cross-project lessons: `logs/lessons.md`
- Common pitfalls and fixes: `logs/pitfalls.md`
- Long-term technical decisions: `logs/decisions.md`
- Coding workflow: `sops/coding.md`
- Debugging workflow: `sops/debugging.md`
- Review workflow: `sops/review.md`
- Communication workflow: `sops/communication.md`
- Project requirements: `projects/<project>/requirements.md`
- Project tech stack: `projects/<project>/tech-stack.md`
- Project decisions: `projects/<project>/decisions.md`
- Project development log: `projects/<project>/dev-log.md`
- Project pitfalls: `projects/<project>/pitfalls.md`
- Project-specific SOP: `projects/<project>/sop.md`
- Project review notes: `projects/<project>/review.md`

## New Project Rule

Create each new project under `projects/` with a clear English or pinyin short name, such as `projects/my-app/` or `projects/data-cleaning-tool/`.

Start from `projects/_template/` and fill only what is known.

## Update Routing

- Durable personal preference: `preferences.md`
- Reusable cross-project lesson: `logs/lessons.md`
- Repeated problem and fix: `logs/pitfalls.md`
- Long-term technical choice: `logs/decisions.md`
- Repeatable workflow: `sops/`
- Project-only information: `projects/<project>/`
